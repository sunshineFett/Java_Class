Object-Oriented Programming

Lecture 2

Lecture Outline

 

I.                    Structured Programming vs. Object-oriented Programming

 

a.       Structured Programming

                                                               i.      Definition – procedure or action-oriented

                                                             ii.      Key Terms – “data processing”

                                                            iii.      Example – Report generation

b.      Definition of Object-Oriented Programming

                                                               i.      Definition – Design emphasizing the view that software systems model things in the real world.

                                                             ii.      Key Terms - “reusability and modularity”

                                                            iii.      Example – Report generation continued

II.                 Fundamental Concepts of OOP

a.       Objects

                                                               i.      Definition – 3 fundamental concepts:

1.      Contain Data

2.      Have Set of Defined Behaviors

3.      Have Individual Identity

                                                             ii.      Example – a compact car

b.      Classes

                                                               i.      Definition - blueprints used to create objects

                                                             ii.      Parallel analogy:  objects/classes as  primitives/variables

                                                            iii.      Relationship between Classes – 3 types:

1.      uses

2.      containment (“has-a”)

3.      inheritance (“is-a”)

c.       Encapsulation

                                                               i.      Key Terms – “data hiding”

                                                             ii.      Definition – 2 distinct parts:  the interface and the implementation

                                                            iii.      Example – the automobile

d.       Inheritance

                                                               i.       Key Terms – base class, superclass, subclass

                                                             ii.      Definition – from general to specific

                                                            iii.      Advantages

1.      Practical – no duplication of method declaration

2.      Templates – abstract out commonalities

                                                           iv.      Uses the “extends” keyword

e.       Polymorphism

                                                               i.      Definition – same message to different objects

                                                             ii.      Example – doubleClick() method

III.               Java Program Basics

a.       Scope – No globals!

b.      Naming conventions – use nouns as Class names; initial capital letter

c.       Multiple classes in a single file

                                                               i.      Valid identifiers

                                                             ii.      Case-sensitive

                                                            iii.      Filename must match the name of the public class

                                                           iv.      Design guideline:  application classes and helper classes

d.      Main method

                                                               i.      Syntax:  public static void main(String[] args)

                                                             ii.      Test your classes with a main method

                                                            iii.      Each time main is called, a thread is spawned

e.       Braces – used as delimiters for classes, methods, loops.

f.        Whitespaces – Java ignores these

g.       Java Coding Conventions

                                                               i.      Each statement on a single line, if possible

                                                             ii.      Indent lines of code within control flow statements

                                                            iii.      Coding conventions for identifiers

1.      Mixed case starting with capitals for classes and packages

2.      Mixed case starting with lower-case for fields and methods

3.      Contants are in all caps

4.      Class definition follows accessibility:  most public to most private; also follows this declaration order:  constants, constructors, methods, static methods, instance variables, static variables.

IV.              Comments

a.       2 ways - // or /* */

V.                 Example:  “Hello World”
