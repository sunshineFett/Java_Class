Methods & Parameters

Lecture 4

Outline

 

I.                    Methods

a.       Design Guidelines:

                                                               i.      Design your methods to represent a single operation. 

                                                             ii.      Limit the method argument list to a reasonable amount. 

                                                            iii.      Methods for testing, debugging and printing out the state. 

b.      Documenting Methods:

                                                               i.      Documenting your code is a thankless task

                                                             ii.      Java comes with a tool called javadoc which creates HTML documentation for your classes.  Using javadoc is a 2-step process:

1.      The first step is to use a special comment symbol to mark the beginning of each method and class.  A javadoc comment begins with the symbol /** and ends, like a regular multilane comment, with */. 

2.      In addition, there are several special tags that can be used inside a javadoc comment

3.      Javadoc reference found at http://java.sun.com/javadoc

c.       Syntax of Methods:

                                                               i.      <object reference>.<method name>(<parameter list>)

                                                             ii.      <class name>.<static method name>(<parameter list>)

                                                            iii.      <method name>(<parameter list>)

d.      Types of Methods:

                                                               i.      Instance method – can have any access modifier; multiple copies get created each time a new object of the class is instantiated.  In order to use instance methods and variable, you must have an object.

 

                                                             ii.      Class (or static) method – belongs to the class itself; singleton pattern

                                                            iii.      Abstract method – a method without implementation; contains the method signature only.

                                                           iv.      Overloaded method

                                                             v.      Overridden method

 

II.                 Parameters

a.       Parameter Passing – Objects communicate with each other by passing messages. 

 

Syntax:  <object reference>.<method name>(<parameter list>) 

   //Explicit object reference

                      

  <class name>.<static method name>(<parameter list>)

                        // Class name

                      

 <method name>(<parameter list>)

   // Object or class is implicitly implied

 

                                                               i.      Passing Primitives – In Java, all parameters are passed by value.  This means that a copy of the parameter is made when that variable is passed to the method.  However, in the case of primitives, any changes to the copy are NOT reflected in the original.

 

                                                             ii.      Passing Object References – In Java, all parameters are passed by value.  This means that a copy of the parameter is made when that variable is passed to the method.  This means that there will be two object references pointing to the same memory location.  Therefore, to alter the copy in the method will, subsequently, alter the original object reference as well.
