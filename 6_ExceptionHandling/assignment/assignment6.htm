Exceptions and Error Handling

Assignment for Lecture 6

 

 

  1. Create your own custom exception class called ChatIOException that extends from IOException. HINT:  you will need to import the java.io package in order to extend from IOException.
    Also, place the ChatIOException class within the following package name: javatraining.example.multithreads.chat

 

  1. Be sure to declare 2 constructors for the ChatIOException class.

 

  1. The first is the default constructor for the ChatIOException class.  This constructor takes in no parameters and calls super() within the implementation block.

 

  1. The second constructor takes in a string as a parameter and passes this string into the call to super();

 

  1. You cannot “run” this class.   Just write the code and compile it but do not try to run it, it will fail if you do.
